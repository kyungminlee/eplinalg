# Test suite map

A comprehensive map of every test family in this repo: what it tests,
how it tests it, where it lives, and how to run it.

Two distinct comparison schemes underlie everything:

- **Scheme A â€” overlay vs migrated** (`tests/blas_parallel/`): the C
  overlay project. Each test calls *both* `<name>_` (overlay) and
  `<name>_migrated_` (migrated Fortran reference at the same kind),
  then either compares (fuzz) or times (bench/perf).
- **Scheme B â€” migrated vs Netlib reference compiled at
  REAL(KIND=16)** (`tests/{blas,lapack,blacs,pblas,pbblas,scalapack,mumps,xblas,ptzblas}/`):
  differential precision. Run the same operation through the migrated
  extended-precision library and through Netlib source compiled with
  `-freal-8-real-16`, compare in `REAL(KIND=16)`. JSON reports under
  `<build>/precision_reports/`, aggregated by
  `scripts/precision_report.py`.

**Critical asymmetry:** the overlay only exists for BLAS today. Every
other library (lapack/blacs/pblas/pbblas/scalapack/mumps/xblas/ptzblas)
is migrated-only and tested only against the Netlib reference.

## Scheme A: overlay vs migrated (parallel-blas)

Lives under `tests/blas_parallel/`. One shared driver pattern:

```
fypp template (consistency/fuzz_*_body.fypp or bench/bench_*_body.fypp)
       â”‚
       â”‚ #:include  via per-target shim consistency/target_<name>/fuzz_<prefix><routine>.fypp
       â–¼
generated .f90 (or .c/.cpp for perf â€” fully script-generated)
       â”‚
       â–¼
linked against both ${LIB_PREFIX}blas (overlay) and ${LIB_PREFIX}blas_migrated
       â”‚
       â–¼
binary tests/blas_parallel/{fuzz,bench,perf}_<routine>
```

Three flavors of test:

| Subdir | Purpose | Driver | Coverage | Total binaries |
|---|---|---|---|---|
| `consistency/` (a.k.a. fuzz) | Correctness equivalence. 200 random cases per binary by default; max relative error must clear tolerance | Fortran `.fypp` | 75 routines Ã— 3 targets | 225 |
| `bench/` | Wall-clock perf, Fortran-side timing (`system_clock`). Emits JSON to `BLAS_BENCH_OUT` | Fortran `.fypp` | 74 routines Ã— 3 targets | 222 |
| `perf/` | Kernel-isolated C perf harness â€” built per-executable with `--gc-sections`, `clock_gettime` timing. Higher fidelity than bench/. | C/C++, generated by `scripts/gen_perf_harnesses.py` | 65 routines Ã— 3 targets | 195 |

Plus shared infrastructure:

- `common/fuzz_util_body.fypp` â€” the kind10/kind16 fuzz_util module (rk-parameterized: `rand_int_log`, `rand_trans`, `rand_alpha_beta`, `fill_matrix_*`, `rand_pad`, `rand_incx`, `rand_incx_pos`, `err_stats_*`)
- `common/target_<name>/fuzz_util.fypp` â€” per-target shim that sets `rk_value`
- `common/target_multifloats/fuzz_util.fypp` â€” standalone module (DD types aren't a numeric KIND, so it's hand-written, not a body shim). Carries the same public surface
- `perf/perf_common.h` â€” `perf_emit`, `perf_emit_json`, `perf_env_int`, `perf_parse_int_list` used by every generated `perf_*.c`

Targets:

- `kind10` â€” `long double` / `_Complex long double` (80-bit on x86-64)
- `kind16` â€” `__float128` / `_Complex __float128`
- `multifloats` â€” `type(real64x2)` / `type(cmplx64x2)` (double-double)

### Stride / categorical coverage (post 2026-05-18 sweep)

Both `fuzz` and `perf` exercise non-unit strides:

|---|---|---|---|---|
| Fuzz | `{1, 2, 3, -1, -2}` weighted 50/20/15/10/5 | independent | `{N, U}` | `rand_trans()` â†’ N/T/C |
| Fuzz scal/iamax/norm (pos-only per BLAS spec) | `{1, 2, 3}` | â€” | â€” | â€” |

## Scheme B: migrated vs Netlib reference @ KIND=16 (differential precision)

Tests the *migrated* libraries against the original Netlib source
compiled with `-freal-8-real-16`. All comparisons happen in
`REAL(KIND=16)` regardless of which migrated target is under test. The
testing model is documented in detail in `tests/README.md`.

Test data is generated at `KIND=16`, converted down to the target's
precision before calling the migrated routine, converted back to
`KIND=16` after, and compared against the reference call which stayed
at `KIND=16` throughout.

```
tolerance = safety_factor * n_ops * target_eps
pass      = max_rel_err <= tolerance
```

The reference is **not** an oracle: `max_rel_err = |ref âˆ’ got| / |ref|`
naturally returns ~0 when both sides diverge from "truth" in the same
way. The test fires only when the migrated routine diverges from the
reference *algorithm*.

|---|---|---|---|
| `tests/blas/` | 25 L1 + 33 L2 + 17 L3 = **75** | `refblas_quad` (Netlib BLAS @ KIND=16, built from `_refblas_src/`) | Canonical scheme; mirrored by every other domain |
| `tests/xblas/` | 15 L1 + 14 L2 + 15 L3 = **44** | Netlib XBLAS @ extended precision | Tests the `BLAS_qgemv_x` etc surface |
| `tests/lapack/` | **669** under `auxiliary/`, `eigenvalue/`, `factorization/`, `linear_solve/` | `reflapack_quad` | Largest domain |
| `tests/ptzblas/` | **35** | tz-prefix triangle/packed-zero BLAS variants | |

| `tests/pbblas/` | **21** | PBBLAS ref | Distributed pbblas |
| `tests/scalapack/` | **178** under `auxiliary/`, `eigenvalue/`, `factorization/`, `linear_solve/` | serial quad LAPACK on global matrix | Distributed |
| `tests/mumps/` | **25** under `fortran/` and `c/` | `refmumps_quad` (sparse direct) | Both Fortran and C entrypoints |

Plus infrastructure:

- `tests/blas/refblas/` â€” builds `refblas_quad.a` from `_refblas_src/`

## How to run

| Test type | Command |
|---|---|
| All overlay fuzz tests (200 cases each via CTest) | `ctest -L blas_parallel` (or `ctest -R 'fuzz_'`) |
| Single overlay fuzz | `./build/tests/blas_parallel/fuzz_egemv` â€” env: `BLAS_FUZZ_CASES`, `BLAS_FUZZ_SEED` |
| Single overlay perf | `BLAS_PERF_SIZES=1024 BLAS_PERF_ITERS=200 ./build/tests/blas_parallel/perf_egemv` â€” env: `BLAS_PERF_WARMUP`, `BLAS_PERF_INCX`, `BLAS_PERF_INCY`, `BLAS_PERF_JSON`, `BLAS_PERF_LABEL` |
| Single overlay bench (Fortran) | `BLAS_BENCH_OUT=foo.json ./build/tests/blas_parallel/bench_egemv` â€” env: `BLAS_BENCH_ITERS`, `BLAS_BENCH_WARMUP`, `BLAS_BENCH_SIZES` |
| Regenerate perf harnesses after editing emitters | `uv run scripts/gen_perf_harnesses.py` |
| Differential precision (BLAS) | `ctest -R '^blas_'` â€” writes JSON to `precision_reports/` |
| Aggregate precision JSON â†’ Markdown | `uv run scripts/precision_report.py` |

## Quick mental model

- **fuzz** = randomized correctness, overlay vs migrated, BLAS only, `tests/blas_parallel/consistency/`
- **bench** = Fortran wall-clock perf, overlay vs migrated, BLAS only, `tests/blas_parallel/bench/`
- **perf** = C kernel-isolated perf, overlay vs migrated, BLAS only, `tests/blas_parallel/perf/` (script-generated)
- **precision tests** = migrated vs Netlib@KIND=16, all 9 library domains, JSON-aggregated via `precision_report.py`
