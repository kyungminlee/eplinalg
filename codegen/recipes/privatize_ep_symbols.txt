# Symbol-privatization manifest for the extended-precision linear-algebra
# stack (task 44: m/w-vs-MKL BLACS ABI collision, np>=2 SIGSEGV).
#
# One exact linker-level symbol name per line. Every migrated staging
# whose recipe carries ``privatize_symbols: privatize_ep_symbols.txt``
# has each of these names renamed to ``ep_<name>`` -- definitions, call
# sites and headers uniformly -- at the end of the migration pass
# (codegen/migrator/privatize.py). The nm audit gate reads the same file, so
# the rename pass and the gate share one source of truth.
#
# Contents: the union of the family-independent support symbols defined
# by blacs_common, pblas_common, scalapack_common (incl. the TOOLS
# helpers now hosted by scalapack_tools) and ptzblas_common -- i.e.
# everything the Netlib-derived engine exports under the same names as
# MKL's proprietary BLACS/PBLAS/ScaLAPACK internals while disagreeing on
# struct layout (BLACBUFF 48 vs 56 bytes). xblas has no collision and is
# excluded. Full analysis and the regenerator:
#   workspace/task/44-mw-blacs-mkl-buffer-overrun-20260709/
#     01-findings.md, 04-implementation-research.md (sec. 1a),
#     research/audit.sh -> research/out/def_*_common.txt
#
# Entries are decoration-exact: ``blacs_gridinit_`` (Fortran-callable)
# and ``Cblacs_gridinit`` (C-callable) are separate lines, as are the
# ``reshape`` / ``reshape_`` / ``RESHAPE`` alias spellings emitted by
# the TOOLS C sources. Do not add patterns; add exact names.
